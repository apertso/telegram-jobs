# Инструкция браузерному агенту — сбор вакансий из Telegram Web

## Цель
Последовательно открыть указанные источники в Telegram Web, собрать последние
сообщения и отправить их локальному обработчику для извлечения подходящих
вакансий. Браузер уже открыт с активной сессией Telegram Web в текущем профиле
пользователя Google Chrome — новый браузер не запускать.

## Источники (обрабатывать строго в этом порядке)
{SOURCES}

## Параметры
- Количество последних сообщений на каждый источник: {COUNT}
- Локальный endpoint для отправки сообщений: {ENDPOINT}

## Порядок работы для каждого источника
1. Откройте ссылку источника (перейдите по ней в текущей вкладке).
2. Дождитесь загрузки сообщений. При необходимости прокрутите чат вниз, чтобы
   подгрузить последние сообщения. Последние сообщения находятся внизу чата.
3. Не собирайте служебные элементы интерфейса, разделители дат и системные
   сообщения — только сами сообщения с текстом.
4. Вызовите действие `collect_and_submit` БЕЗ параметров. Оно само соберёт последние
   сообщения из открытого чата/ветки (лимит — {COUNT}), отправит их на endpoint
   {ENDPOINT} и вернёт статистику. Источник определяется автоматически по текущей
   открытой странице.
5. Перейдите к следующему источнику из списка.

## Важно
- Обрабатывайте источники последовательно, в указанном порядке.
- Если конкретный источник не загружается, выдаёт ошибку или сообщения не найдены —
  зафиксируйте это и переходите к следующему. Не останавливайтесь и не повторяйте
  попытки бесконечно.
- Для ссылок с параметром `thread` собирайте сообщения именно из указанной ветки
  (она уже открыта по ссылке).
- После обработки всех источников завершите работу.
